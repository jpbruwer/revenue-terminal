# revenue-terminal

> ⚠️ **Still in testing** — this is a fun side project, not production software. Expect rough edges.

A terminal-style revenue analytics dashboard built entirely in vanilla HTML/JS with no frameworks or dependencies. Started as a personal experiment to see how far pure browser tech could go with quantitative finance concepts.

---

## What it does

- **OLS Regression** — ordinary least squares trend fitting on revenue data
- **Multi-model Forecasting** — compares multiple forecast models side by side
- **Monte Carlo Simulation** — probabilistic revenue projections
- **Time-Series Decomposition** — breaks revenue into trend, seasonal, and residual components
- **Quantitative Finance Signals** — momentum and volatility indicators

---

## Stack

| Layer | Choice |
|-------|--------|
| UI | Vanilla HTML / CSS / JS |
| Charts | Pure canvas / SVG |
| Math | Hand-rolled (no libraries) |
| Dependencies | None |

---

## Why

Wanted to build something that looked like a Bloomberg terminal but ran in a browser tab. Also an excuse to implement Monte Carlo and OLS from scratch without leaning on numpy or any math library.
